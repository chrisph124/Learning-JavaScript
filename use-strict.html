1/ Chế độ Strict Mode là gì ?

Trước tiên, khi làm việc với Javascript, bạn có thể sử dụng biến mà chưa cần định nghĩa trước(hoisting), có thể quên đặt dấu chấm phẩy ở cuối mỗi đoạn code, có thể sử dụng tên 1 từ khóa để làm tên biến.

Điều này không sai nhưng sẽ thiếu sự ổn định và an toàn. Và từ phiên bản 1.8.5 của Javascript trở đi đã bổ sung 1 khái niệm gọi là Strict Mode để ngăn chặn, giải quyết việc này.

2/ Use Strict là gì ?

Use Strict là từ khóa để khai báo kích hoạt chế độ Strict Mode, bạn muốn sử dụng Strict Mode ở đâu thì chỉ cần đặt từ khóa Use Strict ở đó.

Toàn cục(global) và cục bộ(local) là 2 phạm vi để sử dụng.

Toàn cục(global) là khi bạn đặt use strict ở ngoài 1 hàm(function) và phía trên cùng của 1 file script thì lúc này tất cả các đoạn code bên dưới đều chịu ảnh hưởng của strict mode.

Mặt khác, cục bộ(local) nghĩa là bạn đặt use strict ở trong 1 hàm(function) nào đó và phạm vi ảnh hưởng chỉ gói gọn bên trong hàm đó mà thôi.

a/ Ví dụ về tính toàn cục(global)

// Phía trên cùng của file

"use strict";

// Đoạn code này lỗi vì biến domain chưa được khởi tạo

domain = "http://freetuts.net";

// In ra màn hình

document.write(domain);

Ở ví dụ trên, code sẽ báo lỗi vì khi đã gán use strict lên trên cùng đoạn script thì code bên trong sẽ phải tạo biến domain trước rồi mới được sử dụng, không thể sử dụng trước như bình thường được.

Điều kiện này vẫn đúng(vẫn báo lỗi) cho dù bạn sử dụng 1 biến chưa khởi tạo bên trong 1 hàm nào đó mà vẫn nằm dưới use strict

b/ Ví dụ về local(cục bộ)

//Tạo 1 hàm

function do_sth(){

"use strict";

}

domain = "Let's do something";

document.write(domain);

Ở trên đây thì vẫn đúng thì khi in ra ta không chạy hàm do_sth() mà chỉ in ra biến domain, mặc dù chưa khởi tạo biến nhưng như đã nói ở trên, javascript được quyền chạy 1 giá trị biến dù biến đó chưa được khởi tạo.

c/ 1 số trường hợp bị lỗi, không hợp lệ khi chạy chế độ Strict Mode

  1. Sử dụng biến chưa được định nghĩa
  2. Không chấp nhận Delete biến
  3. Định nghĩa 1 thuộc tính 2 lần
  4. Khai báo tham số bị trùng(vd : khai báo 2 tham số trùng tên trong 1 hàm)
  5. Lỗi literals và escape với number(không được sử dụng literals và escape với kiểu number)
  6. Khai báo tên biến trùng với key(trùng tên hàm)

1 số từ khóa không được đặt làm tên biến

  1. implements
  2. interface
  3. package
  4. private
  5. protected
  6. public
  7. static
  8. yield
